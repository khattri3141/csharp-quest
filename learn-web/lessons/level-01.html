Hello, World!

Run your very first C# program and print text to the screen.

i What you'll learn

  • What a C# program looks like.
  • What Console.WriteLine does.
  • How to run code with dotnet run.

Lesson plan (≈ 15 min)

  • 2 min — Open VS Code on the Level 1 folder side-by-side with this page.
  • 5 min — Walk through the concept + the line break-down together.
  • 5 min — Student types Console.WriteLine("Hello, World!"); and runs dotnet run.
  • 3 min — Run dotnet test together. Then take the quiz.

Before you start, ask

"When you open an app on your phone, what is the very first thing the computer has to do?"

Goal of the question: get them to say "follow instructions one by one." That's literally what a program is.

1 The concept

A C# program is a set of instructions the computer follows in order. The simplest instruction is: show some text on the screen.

Real-world analogy: Think of Console.WriteLine("Hello") as you handing a sticky note that says "Hello" to a friend and asking them to read it out loud. Console is the friend. WriteLine is the action. The text in quotes is the note.

Here is the smallest C# program possible (.NET 8 style):

// Program.cs
Console.WriteLine("Hello, World!");

That's the entire file. .NET 8 lets you skip the boilerplate class and Main — this is called top-level statements.

Why this matters: Every app you'll ever build starts with running some first instruction. Today that instruction is "print a string." Next week it could be "start a web server."

Talk through it

  • Read the one line of code out loud, left to right: "Console — dot — WriteLine — open paren — quote — Hello, World — quote — close paren — semicolon."
  • Make sure they hear the structure: a thing (Console), a verb (WriteLine), an argument (the string), a terminator (the semicolon).

Check understanding — ask

"What would happen if I forgot the semicolon at the end?"

If they don't know, that's fine — let them try it in the next step and see the error.

2 Break it down

Let's read the one line you'll write, piece by piece:

  • Console — a built-in object that represents the terminal window.
  • .WriteLine(...) — a method (function) that writes text and moves to the next line.
  • "Hello, World!" — a string: text wrapped in double quotes.
  • ; — every C# statement ends with a semicolon. Forget it and the compiler complains.
Watch out: C# is case-sensitive. console.writeline will not compile. Capital C, capital W, capital L.
Interview answer: "In .NET 8, top-level statements let a console app skip the explicit Main method — the compiler generates it for you. Console.WriteLine writes a line to standard output."

Common student mistakes — watch for

  • Lowercase console.writeline — case matters; "case-sensitive" is a real concept worth naming out loud.
  • Single quotes 'Hello' — in C#, single quotes are for a single character (char), not a string.
  • Missing the semicolon — show the compile error rather than fixing it for them.
  • Smart quotes from auto-correct (“ ”) — happens if they paste from chat or docs.

> Console challenge

Time to write real code. Open this folder in VS Code:

practice-console/Level01-HelloWorld/

  1. Open the folder in VS Code.
  2. Open Level01/Program.cs. You'll see a TODO comment.
  3. Replace the TODO with a line that prints Hello, World! exactly.
  4. Open a terminal in that folder and run: 
    cd Level01
dotnet run
    You should see your message printed.
  5. Now prove your code is correct — from the parent folder run: 
    dotnet test
    All tests should be green.
Tip: If dotnet is not found, install the .NET 8 SDK from dotnet.microsoft.com/download and restart your terminal.

Make it stick

  • Before they hit Enter on dotnet run, ask: "what do you expect to see?" Predicting first builds the mental model.
  • After it works, ask them to change the message to their name. Then run again. Tiny wins keep momentum.
  • Now run dotnet test together — explain that tests are "small robots that check your work."
