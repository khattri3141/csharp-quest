Console I/O

Read user input. Talk back to them. Convert text into numbers.

Warm-up question

"If I type my age into the terminal as text, is that the same as the number 25? What if I tried to add 1 to it?"

Everything that comes in from the keyboard is a string. That is the fundamental I/O wall.

1 Printing output

Two methods print to the terminal. The only difference is whether a newline is added at the end:

Console.Write("Your name: ");    // cursor stays on same line
Console.WriteLine("Hello!");        // cursor moves to next line
Analogy — texting vs sending: Console.Write is like typing a message without pressing Send — the cursor waits on the same line for more text. Console.WriteLine presses Send, moving to the next line.

You can also use string interpolation (the $ prefix) to embed variables directly:

string name = "Sam";
int age = 25;
Console.WriteLine($"Hello, {name}! You are {age} years old.");

Show the difference live

Type both lines in a scratch file and run dotnet run. Have the student predict what the output looks like before running. The cursor-on-same-line effect is memorable when they see it for real.

2 Reading input — Console.ReadLine()

To read a line the user types, call Console.ReadLine(). It pauses the program, waits for Enter, and returns everything typed as a string.

Console.Write("Your name: ");
string? input = Console.ReadLine();   // could be null if the stream closes
string name = input ?? "stranger";    // ?? gives a default if null
Console.WriteLine($"Hello, {name}!");
Why string? with a question mark? The ? means the value can be null. Console.ReadLine() returns null when there is no more input (e.g., a file stream ends). The ?? operator (null-coalescing) gives a safe fallback so your program never crashes.
Interview answer: "Console.ReadLine() returns string? — a nullable string — because input can be absent when the stream is redirected or closed. You handle it with the ?? operator to provide a default value, avoiding a NullReferenceException."

The nullable question mark

Don't go deep on nullability yet. Just say: "The ? means the value might be null. We add ?? "" as a safety net. We'll study nulls properly in a later level."

Ask

"What would happen if you removed the ?? "" and passed null to Greet()?"

3 Converting text to numbers

Everything the user types comes in as a string. If you need a number, you must parse it explicitly. C# will never convert silently.

Console.Write("Enter a number: ");
string text = Console.ReadLine() ?? "0";

int    whole   = int.Parse(text);       // "21"  -> 21
double decimal_ = double.Parse(text);    // "3.14" -> 3.14
Analogy — the currency exchange desk: you walk in with a piece of paper that says "42". It is just text — it cannot be added or subtracted. The int.Parse() desk takes your paper, verifies the number written on it, and hands you back the actual coin worth 42. Now you can do maths with it.
What if the text is not a valid number? int.Parse("hello") throws a FormatException. For now, trust your input. In a later level you'll learn int.TryParse() for safe parsing.

Live crash demo

Deliberately call int.Parse("hello") and let it crash. Read the exception message out loud. Normalise seeing exceptions — they are informative, not scary. Say: "This is the program telling you exactly what went wrong and where."

Key phrase

"The keyboard always gives you text. You choose when to turn it into a number."

4 Putting it all together

Here is a complete mini-program that reads a name and a number, then responds:

Console.Write("Your name: ");
string name = Console.ReadLine() ?? "stranger";

Console.Write("Pick a whole number: ");
string text   = Console.ReadLine() ?? "0";
int    number = int.Parse(text);

Console.WriteLine($"Hello, {name}! Double your number is {number * 2}.");
Flow: prompt → wait → read string → parse → compute → print. Every console app you write follows this same loop.

Trace through it together

Cover the code and ask the student to describe what each line does in plain English, in order. If they can narrate the code, they understand it. This is the most important comprehension check in the level.
